# Politica de Seguranca

## Suporte

Este projeto ainda nao possui SLA formal de seguranca, mas relatos responsaveis sao bem-vindos.

## Como reportar

Se voce identificar uma vulnerabilidade, problema de privacidade ou abuso potencial:

1. nao publique a exploracao em issue publica imediatamente;
2. descreva o problema com passos claros de reproducao;
3. envie o relato ao mantenedor do repositorio pelos canais disponiveis no perfil do GitHub.

## Escopo esperado

Relatos uteis incluem:

- exposicao indevida de dados;
- uso excessivo de permissoes;
- comportamento inesperado em paginas visitadas;
- riscos de injecao de conteudo ou manipulacao insegura de HTML;
- problemas de dependencia ou de integracao externa que afetem usuarios.

## Boas praticas do projeto

O objetivo e manter a extensao:

- com proposito unico;
- com o minimo de permissoes necessario;
- sem backend proprio para coleta de usuarios;
- com comportamento transparente e previsivel.
